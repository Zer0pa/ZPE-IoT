# Support

<p>
  <img src="../.github/assets/readme/section-bars/where-to-go.svg" alt="WHERE TO GO" width="100%">
</p>

| What you need | Where to go |
|---|---|
| General support | `support@zer0pa.com` |
| Security vulnerability | `security@zer0pa.com` and `../SECURITY.md` |
| Audit or evidence replay question | `../AUDITOR_PLAYBOOK.md` and `../PUBLIC_AUDIT_LIMITS.md` |
| Benchmark question | `BENCHMARKS.md` and the cited `validation/results/*.json` artifacts |
| Architecture or package-surface question | `ARCHITECTURE.md` |
| IMC alignment question | `family/IOT_IMC_ALIGNMENT_REPORT.md` |
| Repo access or private issue follow-up | Use the private GitHub repo if you have access, otherwise email `support@zer0pa.com` |

<p>
  <img src="../.github/assets/readme/section-bars/what-gets-a-fast-response.svg" alt="WHAT GETS A FAST RESPONSE" width="100%">
</p>

Reports with:

- a commit SHA or bundle identifier
- operating system, Python version, and Rust toolchain
- a reproducible command or minimal sample
- exact artifact paths when disputing a claim

<p>
  <img src="../.github/assets/readme/section-bars/what-does-not-get-a-response-here.svg" alt="WHAT DOES NOT GET A RESPONSE HERE" width="100%">
</p>

- requests to treat historical `release/RC_*` bundles as current authority
- claims disputes without counter-evidence
- requests to imply public package availability before publication is actually executed

<p>
  <img src="../.github/assets/readme/section-bars/response-expectations.svg" alt="RESPONSE EXPECTATIONS" width="100%">
</p>

This is a private-stage source repository.

- General support requests are acknowledged within 2 business days when possible.
- Security reports are prioritized under `../SECURITY.md`.
- Evidence disputes should cite exact files and exact dates.
