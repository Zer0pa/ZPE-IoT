# CLI Contract

Date: 2026-02-19  
Scope: `python/zpe_iot/cli.py`

## Commands

- `zpe-iot compress <input.csv> --output <out.zpk>`
- `zpe-iot decompress <input.zpk> --output <out.csv>`
- `zpe-iot info <input.zpk>`
- `zpe-iot benchmark <input.csv> [--compare zstd,lz4,zlib]`
- `zpe-iot diagnostics`

## Scriptability

- Each command supports `--json` for machine-readable output.
- JSON payload includes `command` and `exit_code`.
- CSV input parsing supports chunked mode via `--chunk-size`.

## Exit Codes

- `0`: success.
- `1`: runtime/data error (malformed packet, unknown comparator, empty numeric input, invalid runtime option values).
- `2`: usage/argument error generated by Click (missing required args/options, invalid option format).
- `130`: interrupted execution (`SIGINT`).

## Determinism Notes

- Runtime-data failures are surfaced as deterministic `RuntimeDataError` (`exit_code=1`).
- Usage failures are delegated to Click parser semantics (`exit_code=2`).
- `diagnostics` exposes native acceleration availability via `zpe_iot._native.available()`.

## Examples

```bash
zpe-iot compress data.csv --output data.zpk --chunk-size 50000 --json
zpe-iot info data.zpk --json
zpe-iot decompress data.zpk --output restored.csv --json
zpe-iot benchmark data.csv --compare zlib --json
zpe-iot diagnostics --json
```
