# Support Policy

The reader-facing routing guide lives in `docs/SUPPORT.md`. This root file defines the repo-level support boundary and contact surface.

<p>
  <img src=".github/assets/readme/section-bars/where-to-go.svg" alt="WHERE TO GO" width="100%">
</p>

| Need | Route |
|---|---|
| Technical support | `support@zer0pa.com` |
| Security reports | `security@zer0pa.com` |
| Reader-facing routing | `docs/SUPPORT.md` |
| Repo access or private-lane coordination | Use the private GitHub repo if you have access, otherwise email `support@zer0pa.com` |

<p>
  <img src=".github/assets/readme/section-bars/scope.svg" alt="SCOPE" width="100%">
</p>

Support covers:

- API and CLI usage issues
- Rust, Python, and C surface integration questions
- benchmark reproduction assistance
- bug triage with a minimal reproducible case

<p>
  <img src=".github/assets/readme/section-bars/verification.svg" alt="VERIFICATION" width="100%">
</p>

- commit SHA or release bundle identifier
- operating system, Python version, and Rust toolchain
- command or API call used
- logs, stack traces, or failing artifact paths
- sample input if it is safe to share

<p>
  <img src=".github/assets/readme/section-bars/response-expectations.svg" alt="RESPONSE EXPECTATIONS" width="100%">
</p>

- Acknowledge support requests within 2 business days.
- Provide triage status within 5 business days.
